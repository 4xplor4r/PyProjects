Файл - описание, помошник по проекту "Bigram language model".

Простой скрипт, который на основе заданных текстов генерирует новые, построен на запоминании продолжения фразы после каждого уникального слова в тексте.
Алгоритм состоит из обучения и генирации псевдослучайной последовательности продолжений фраз, по заданному префиксу.
Информация в модели хранится в виде словаря.

Интерфейс
Основной код разбит на две части: обучение и генерация.

Обучение:

Считываются входные данные из ".txt" файла или потокового ввода stdin.
Чистятся тексты: выкидываются неалфавитные символы, приводится к lowercase.
Тексты разбиваются на слова.
Модель сохраняется в ".pkl" формате, в корень диска.

Генерация:

Загружается модель.
Генерируется последовательность нужной длинны.

Детали реализации:

Интерфейс реализован через `argparse`.
Для работы с текстами используется библиотека регулярных выражений `re`.
Код соответствкет принципам ООП и стандартам PEP8.
Для сохранения модели используется `pickle`.

Функционал:

Для инициализации и обучения модели потребуется файл "your_file.txt"
Чтобы обучить модель с помощью это файла необходимо написать в терминале указание интерпритатору прочитать и 
исполнить файл, по истинному пути, так же оставить пустым конец строки, что бы активировать потоковый ввод:
>python c:/.../train.py
или кузать дополнительный аргумент "--path your_file.txt":
>python c:/.../train.py --path your_file.txt
Программа автоматически сохранит полученную модель в корне диска, в ".pkl" формате.

Для генерации последовательности необходимо написать в терминале указание интерпритатору прочитать и 
исполнить файл, по истинному пути, так же обязательно указать длинну последовательности ,и 
обязательно использовать аргумент "--path your_file.txt":
>python c:/.../generate.py length_of_sequence --path your_file.txt

Была задуманна возможность обучить модель на стадии генерации, указать необязательный аргумент через `argparse`.
Этим аргументом должен был стать "--learn", который через класс Education, доджен был обучить новую модель. 
Однако возникла проблема с import-ом. Функции `argparse`, смешиваются в ходе логикики.
Поэтому единственной возможностью сгенерировать последовательность, остается по отдельности обратиться
за результатом к каждому из исполняющих файлов.

Так же можно добавить префикс к генерируемой последовательности.
>python c:/.../generate.py length_of_sequence --path your_file.txt --prefix your_prefix
